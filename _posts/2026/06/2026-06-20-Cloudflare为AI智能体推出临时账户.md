序言：AI 智能体可以写代码，但一旦需要部署——需要注册账号、通过 OAuth 验证、复制 API token——它就撞上了一堵为人类设计的墙。Cloudflare 刚刚推出的"临时账户"功能解决了这个问题：任何智能体运行 `wrangler deploy --temporary` 就能在几秒内获得一个可用的 Worker，无需注册、无需登录、无需人工介入。这篇文章展示了智能体时代的部署基础设施应该长什么样。

原文：[Temporary Cloudflare Accounts for AI agents](https://blog.cloudflare.com/temporary-accounts/)（作者：Sid Chatterjee、Celso Martinho、Brendan Irvine-Broque，日期：2026-06-19）

---

如今每个人都在用 AI 智能体写代码。但一旦智能体需要部署什么东西——需要注册和创建账户——它就迎面撞上了一堵为人类设计的墙：基于浏览器的 OAuth 流程、需要点击的仪表盘、需要复制粘贴的 API token、需要满足的多因素认证提示。对于坐在开发者旁边的交互式副驾驶，这只是烦人；对于后台智能体来说，这意味着硬性停摆。

智能体现在可以立刻部署网站、API 和智能体，不需要先注册账户。

任何智能体现在都可以运行 `wrangler deploy --temporary` 将 Worker 部署到 Cloudflare。这个临时部署保持活跃 60 分钟，在此期间你可以认领这个临时账户，把它永久变成你自己的。如果你不认领，它会自动过期。

## 为什么无摩擦部署对 AI 智能体如此重要

无摩擦临时账户的重要性比第一眼看上去更深远：

后台 AI 会话没有人在回路中，这正在成为常态。任何需要浏览器、复制粘贴或"60 秒内点这里"的认证步骤，都意味着智能体被卡住，可能选择去别处部署。

试错是智能体的超能力。智能体需要一个紧凑的 编写 → 部署 → 验证 循环。它们需要廉价、可抛弃的部署目标，这样它们可以 curl 自己的输出并判断自己是否做对了。

智能体平台正在构建自己的方式来让代码部署"开箱即用"，无需额外步骤或凭证。人们开始期望这个流程就是能直接运行的，不需要去注册任何他们没用过或没听说过的其他服务。

临时账户围绕 Wrangler 构建——这是我们开发者平台的命令行工具，让开发者引导新项目、管理配置和资源、以及部署和更新它们。

Wrangler 的用法在网上有广泛的文档记录，智能体也很会用它。但如果你还没有登录并授权 Wrangler 访问你的 Cloudflare 账户，当智能体尝试部署时，它就会卡在注册和认证这一步。你可能会问：智能体和 LLM 怎么知道 Wrangler 这个新的 `--temporary` 标志存在，以便他们在没有人类明确告知的情况下就去用它？

为了解决这个问题，我们更新了 Wrangler，使其在输出中提示智能体关于 `--temporary` 标志的信息。

当智能体发现这一点，然后再次运行 `wrangler deploy --temporary`，Cloudflare 就会为智能体提供一个临时账户、给 Wrangler 一个 API token 来工作，并提供一个认领 URL，让智能体可以传回给人类。

确保你在使用最新的 Wrangler 版本，启动你最喜欢的编程智能体，用构建模式写一个部署 "hello world" 应用的提示词：

> 做一个非常简单的 hello world Cloudflare Worker，用 TypeScript，用 wrangler 部署它，不要问我问题，尽你所能做到最好。

智能体将运行 wrangler，从输出消息中获取 `--temporary` 标志，构建你的脚本，然后立即部署它，无需人参与。

如你所见，智能体写了脚本，使用 `--temporary` 标志部署了它，curl 了输出中拿到的预览链接，并验证了结果与代码匹配。

这很棒，但智能体编码通常不是一次部署就完了的。一个会话可以经历多次代码变更的循环。这不是问题：智能体可以迭代 Worker 脚本并按其所需多次重新部署变更（在 60 分钟认领窗口内）。

> 现在把 hello world 改成 "hello cloudflare" 然后重新部署。

观察智能体修改源代码，复用之前创建的临时账户，重新部署一个新版本并重新检查结果。

在任何时候，你都可以认领临时账户，把它永久变成你自己的。当你点击认领链接时，你将被带到一个页面，在这里你可以注册或登录 Cloudflare，然后认领你的 Worker 被部署到的那个临时账户。这包括认领的不只是 Worker，还有数据库和其他绑定资源。

如果你在 60 分钟内未认领这些临时账户，它们将被自动删除。

## 通向无摩擦智能体部署的道路

这只是我们消除智能体注册障碍的方法之一。我们最近与 Stripe 宣布了一项合作伙伴关系，并共同设计了一个新协议，让智能体可以代表其用户配置 Cloudflare——创建账户、启动订阅、注册域名、获取 API token 来部署代码，无需复制粘贴 token 或输入信用卡信息。上个月，我们与 WorkOS 合作推出 auth.md，任何人都可以采纳，让智能体使用现有的标准 OAuth 来配置新账户。

这个领域正在发生大量进展，我们很兴奋能持续让智能体更容易使用 Cloudflare，也让开发者更容易让自己的应用适配智能体。临时账户是朝着无摩擦智能体部署迈出的又一步——请持续关注更多更新。

临时账户有一些限制，且其能力可能随时间变化；请查阅开发者文档获取更多信息，然后去构建一些东西。把你的智能体指向 Cloudflare，看看它能走多远，告诉我们什么可以改进或什么让你惊喜——在 X 上分享你所构建的，或加入 Cloudflare 社区。
