## 1. 网页净化阅读器 — 一键 AI 重排，告别广告与噪音

**核心功能**：用 AI 大模型将任意网页即时重排为干净的 Markdown 阅读视图，去除所有广告和干扰元素，并可选翻译为中文。

**创意来源**：HN Show 和 V2EX 上几乎同时出现同一个项目 cwsum — 基于 DeepSeek V4 Flash 的 Chrome 插件，调用你自己的 API Key，对网页做 AI 重整。V2EX 网友描述："浏览网页完全不会有广告，可以快速看英文帖子，大模型直接翻译成中文。" 来源：[HN Show](https://news.ycombinator.com/item?id=48728728) / [V2EX](https://www.v2ex.com/t/1223907)

**技术路径**：Chrome 扩展 + DeepSeek V4 Flash API（近乎免费） + Markdown 渲染。核心逻辑不复杂：抓取页面 DOM 主体内容 → 送 LLM 结构化输出 Markdown → 本地渲染阅读视图。可选加入 PWA 支持实现离线缓存。

**差异化优势**：现有的阅读模式（Safari Reader View、简悦等）依赖 DOM 规则提取，复杂页面经常失败；Readwise Reader 等全功能产品太重太贵。这个方案利用 LLM 的语义理解能力，几乎能处理任何页面结构，且 API 成本极低，适合做成一次付费买断的轻量工具。

## 2. 专注卫士 — AI 坐你旁边，帮你关掉分心标签页

**核心功能**：AI 代理实时监测浏览器活动，当你走神打开娱乐/社交标签时自动拦截关闭，帮你保持专注。

**创意来源**：HN Show 上 Drifty 展示了这个概念 — 一个 AI Focus Agent，不是简单的网站屏蔽器，而是动态判断你当前的行为是否属于分心并采取行动。来源：[Drifty](https://drifty.so/) / [HN Show](https://news.ycombinator.com/item?id=48728127)

**技术路径**：Chrome 扩展 + 本地小模型（或廉价 API）做标签页内容实时分类 + 白名单/黑名单规则引擎。可加入番茄钟集成、专注时长统计。关键是要低延迟本机推理，避免隐私问题。

**差异化优势**：现有工具要么是死板的域名屏蔽（Freedom、Cold Turkey），要么是复杂的屏幕时间统计（RescueTime）。Drifty 的思路是有判断力的"智能管家"而非"牢笼看守"——只在真正分心时介入，学习你的工作模式，让专注工具不再令人反感。适合 1-2 周 MVP。

## 3. 离线图片压缩工坊 — 浏览器里批量搞定，不上传服务器

**核心功能**：纯浏览器端离线批量图片压缩，支持实时预览对比、逐张调参、打包下载，零服务器依赖。

**创意来源**：V2EX 上开发者分享了 PicForge，起因是写博客需要批量压缩图片，Google 的 Squoosh 虽好但只能单张处理。开源，支持 PWA 安装。来源：[PicForge](https://picforge.de) / [V2EX](https://www.v2ex.com/t/1223909) / [GitHub](https://github.com/DejavuMoe/PicForge)

**技术路径**：纯前端方案 —— WebAssembly 版图片编解码库（如 Sharp/libvips 编译到 WASM）+ Web Workers 多线程批量处理 + PWA 离线支持。技术栈：React/Vue + WASM + IndexedDB 暂存。

**差异化优势**：市面上图片压缩要么在线服务（隐私风险、文件大小限制），要么桌面软件（需安装、平台绑定）。浏览器 WASM 方案已有成熟基础（Squoosh 证明了可行性），只需在此基础上增加批量队列和项目管理。这本质上是一个"把现有技术组合成独立产品"的机会，开发周期短，变现路径清晰（买断制桌面版或 Web Pro 版）。

## 4. 临期管家 — 拍照即录入，给食材和日用品上"闹钟"

**核心功能**：拍照识别包装上的生产日期和保质期，自动计算临期提醒，在还剩 10% 保质期时推送通知。

**创意来源**：V2EX 开发者分享了一个 Android App ExpiryTrack，自己用了大半年迭代到顺手后上架 Play Store。用户拍物品包装即可自动识别保质期、名称，临期时提醒。来源：[V2EX](https://www.v2ex.com/t/1223769) / [Google Play](https://play.google.com/store/apps/details?id=io.github.twiceyuan.expirytrack)

**技术路径**：Flutter 跨平台（iOS + Android）+ OCR 识别（可用 Google ML Kit 文字识别或 OpenAI Vision API）+ 本地 SQLite 存储 + 系统通知推送。核心识别逻辑：定位包装上的日期文本 → 解析格式 → 计算剩余天数 → 按百分比触发提醒。

**差异化优势**：市场上的冰箱管理 App（如 Fridgly、Pantry Check）功能庞杂，要求手动录入大量信息，用户坚持不下来。聚焦单一功能——"拍照即走、过期提醒"——大幅降低使用阻力。这个品类在国内几乎没有好用的竞品，独立开发者有充分的突围空间。

## 5. 额度看板 — 菜单栏上的 AI 编程助手用量监控

**核心功能**：macOS 菜单栏小组件，实时显示 Codex/Claude Code 等 AI 编程助手的剩余额度、重置倒计时和消费统计。

**创意来源**：V2EX 上开发者分享了 Codex Runway，一个原生 macOS 状态栏应用，让重度 AI 编程用户"少打开网页，少猜额度，少错过重置时间"。来源：[V2EX](https://www.v2ex.com/t/1223799) / [GitHub](https://github.com/Licoy/codex-runway)

**技术路径**：SwiftUI 原生 macOS 应用 + 状态栏小组件 + 各家 API 用量端点聚合。小到可以在一个周末完成 MVP：查阅各家 API 的 usage/billing 端点 → 定时轮询 → 菜单栏展示数值和进度条 → 可配置阈值告警。

**差异化优势**：随着 Cursor、Codex、Claude Code 等工具成为开发者日常高频工具，额度管理成了真实痛点——你可能同时订阅了好几个、各家重置周期不同、超额罚款很贵。这是一个典型的"大厂不屑做、用户真需要"的细分小工具，可以通过 App Store 付费下载或内购高级功能变现。

---

> 今日最佳推荐：**网页净化阅读器** 和 **临期管家**。前者是"技术时机刚好"——廉价 LLM API 让网页语义清洗变得可行；后者是"需求真实且竞争对手少"——国内几乎没有好用的保质期管理 App。两者都具备单点突破、独立友好、极简单功能的特点。
