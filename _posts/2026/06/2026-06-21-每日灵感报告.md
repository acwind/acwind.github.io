## 1. Empty · 空 — AI 阅读伴侣，只聊你读过的内容

**核心功能**：一个尊重剧情边界的 EPUB/PDF 阅读器，AI 只能在读者已读完的文本范围内回答问题，彻底杜绝剧透。

**创意来源**：V2EX 用户 davirain 分享了他用 SwiftUI 原生开发的阅读器 "Empty · 空"。市面上 AI 阅读工具默认读取全书再总结，对未读完的读者来说，"全知视角"本身就是剧透。"空" 内置了 AI 助手"朱"，只在已读范围内帮助翻译、解释、回顾。项目还规划了"分册管理"功能，让用户自由划分剧情边界。来源：[V2EX](https://www.v2ex.com/t/1221737#reply0)

**技术路径**：SwiftUI（iOS/macOS）、EPUB/PDF 解析、Core Data 本地存储、本地 LLM 推理（MLX 或 Ollama），可选择接入 OpenAI 兼容 API。核心逻辑在于维护一个已读位置指针，所有 AI prompt 都注入"你只知道主角读到了第 X 章"的约束。

**差异化优势**：相比 Kindle 的 X-Ray 或 Apple Books 的搜索功能，它不做全局知识图谱，而是刻意限制 AI 的视角范围。这个"限制"本身就是产品价值，是一个反直觉但精准的设计决策。

## 2. LearnPath · 学习路径排序引擎

**核心功能**：输入一个复杂主题，自动生成从入门到精通的"最优阅读和学习顺序"，每步附上简短理由说明为什么要先看这个再看那个。

**创意来源**：Reddit r/SomebodyMakeThis 用户 NovaFire7 分享了他开发的工具，解决"学新东西总是开 40 个标签页却不知道从哪开始"的痛点。目前覆盖了 ML（强化学习、LLM、Agent）、创业（融资、公司文化）等主题，按难度分层排列资源，从简单的入门文章到深度论文形成一条清晰路径。来源：[Reddit](https://www.reddit.com/r/SomebodyMakeThis/comments/1uargg4/i_made_a_tool_that_gives_you_the_ordered_what_to/)

**技术路径**：Next.js 前端 + Python/FastAPI 后端，核心使用 LLM 进行资源排序和理由生成，结合 Reddit/HN/各社区点赞数据做质量加权。可考虑用向量数据库存储资源 embedding，让用户上传自己想学的主题 PDF 后生成定制化路径。

**差异化优势**：不同于 Roadmap.sh 的静态路线图或 Coursera 的课程列表，它聚焦于"具体读哪篇文章/论文/视频"，是一个动态、可论证的学习导航器，每个步骤都有"为什么这个排在前面"的解释。

## 3. Agent Traffic Light · 多 AI Agent 桌面红绿灯

**核心功能**：在桌面角落显示多个 AI 编程 Agent 的实时运行状态，通过红绿灯颜色（黄=运行中、绿=成功、红=失败、灰=已死）一目了然掌握所有会话进度。

**创意来源**：V2EX 用户 makubex1324 分享了他同时开 3-4 个 Claude/Codex agent 干活时遇到的痛点：反复切换终端窗口才发现某个 agent 半小时前就挂了。于是写了一个叫 agent-traffic-light 的 macOS 桌面小工具，每个 agent 一张三色灯卡片，窗口无边框、置顶、可拖拽。来源：[V2EX](https://www.v2ex.com/t/1221606#reply1)

**技术路径**：Electron 或 Tauri + React（跨桌面平台），通过监控 agent 进程的 stdout/stderr 解析状态（运行/成功/失败/退出），WebSocket 用于实时通信，托盘常驻。可扩展支持多种 agent 框架：Claude Code、Codex、Cursor、Gemini CLI 等。

**差异化优势**：相比 macOS 的活动监视器或终端分屏，它将复杂的多进程状态浓缩为极简的视觉信号——一张扑克牌大小的色块。不需要任何交互就能获取信息，符合"扫一眼就知道"的桌面小组件哲学。

## 4. 古人早就说过 · 网络热梗→古诗文反查

**核心功能**：输入一句网络流行语（如"我 emo 了"、"卷不动了想躺平"），反查出表达相同意境的古诗文出处（如"抽刀断水水更流，举杯消愁愁更愁"、"采菊东篱下，悠然见南山"）。

**创意来源**：V2EX 用户 yandong9713 被一个教育类短视频启发：一位父亲教女儿用古诗文对应网络用语。于是用 Next.js 做了 gurensaid.com，采用语义向量匹配算法，让 AI 先理解网络用语的情感内核，再到古诗文数据库中搜索意境相似的句子。开源在 GitHub。来源：[V2EX](https://www.v2ex.com/t/1221609#reply9)，[项目地址](https://gurensaid.com)

**技术路径**：Next.js / Nuxt（SEO 友好，适合内容型工具），向量数据库（Pinecone/Weaviate/Milvus）存储古诗文 embedding，前端极简搜索框 + 卡片式结果展示。古诗文语料可从《全唐诗》《全宋词》公开数据集中导入，用 sentence-transformers 做中英双语向量化。

**差异化优势**：大部分古诗文工具是"输入古诗→查解释"，这个产品反其道而行——"输入白话→找古诗"。它在教育和社交传播两方面都有天然优势：家长用来教孩子，年轻人用来发朋友圈配文。配合短视频传播，有成为现象级小工具的潜力。

## 5. Shelve · 下载文件夹自动整理器

**核心功能**：监控 macOS 下载文件夹，根据文件类型、来源和命名规则自动归类整理，无需手动操作。

**创意来源**：HN Show 用户 danielzx1 使用 Swift/SwiftUI 开发了原生 macOS 菜单栏应用 Shelve，自动按规则整理 Downloads 文件夹。这是那种"每个人都需要但不觉得有人会专门做"的小痛点——下载文件夹永远是桌面上最乱的地方。开源在 GitHub。来源：[HN](https://news.ycombinator.com/item?id=48615915)，[GitHub](https://github.com/DanielZ1-tech/shelve)

**技术路径**：Swift/SwiftUI（macOS 原生）、FileMonitor API 监控文件系统变化、AppKit 菜单栏集成。规则引擎可支持自定义：按扩展名归类（PDF→文档、DMG→安装包）、按来源域名归类（浏览器下载 URL 的 hostname）、按日期归档。

**差异化优势**：相比 Hazel（功能过于强大且付费）和 macOS 自带的"堆栈"功能（只能临时分组），Shelve 追求的是"零配置、自动运转"。用户安装后就忘记它的存在，但下载文件夹永远整洁——这种"隐形工具"的设计理念本身就是一种产品美学。
