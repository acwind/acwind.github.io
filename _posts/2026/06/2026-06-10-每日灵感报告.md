## 1. LLM API 比价看板

**核心功能**：一张实时更新的表格，聚合 40+ 个大语言模型 API 的输入、输出、缓存价格，支持人民币和美元双币种对比。

**创意来源**：V2EX 用户 hnlszcl611 分享了自己做的「算盘」工具，一张表覆盖了 DeepSeek、通义、Kimi、智谱、豆包、文心、混元等 10 家国产厂商，以及 GPT、Claude、Gemini、Grok 等海外模型的价格对比。每个价格都标注了最后核对日期和官方定价页直链，汇率每天自动更新。这个品类的信息差很大——开发者选 API 时经常要在十几个定价页之间跳来跳去。（[来源](https://www.v2ex.com/t/1219163)）

**技术路径**：Next.js 静态生成 + GitHub Actions 每日定时运行脚本抓取定价页 + Vercel 部署。核心是写好几个厂商的定价页爬虫，数据存 JSON，前端渲染表格。不需要后端数据库。

**差异化优势**：现有方案要么是零散的博客文章（过时快），要么是某个厂商自己的价格计算器（只有一家）。一个中立的、自动更新的聚合看板，解决的是「我该用哪个模型最省钱」这个高频决策问题。

## 2. GitHub Release 应用商店

**核心功能**：把 GitHub 仓库的 Releases 页面变成应用商店体验——自动识别平台对应的安装包（.dmg、.exe、.apk），一键下载，并跟踪更新。

**创意来源**：小众软件推荐了 GitHub Store 这个开源项目。很多优秀的开源软件只在 GitHub 发布，普通用户要找到正确的安装包得在 Assets 列表里翻找，更别提后续更新了。这个工具自动检测操作系统并突出显示对应的安装包，还能在安卓上跟踪软件更新。（[来源](https://www.appinn.com/github-store-third-party/)）

**技术路径**：前端用 React/Next.js，通过 GitHub API 获取仓库的 Releases 信息，解析 Assets 文件名判断平台类型。可以做成 PWA，添加到主屏幕后体验接近原生应用商店。缓存策略用 localStorage 或 IndexedDB。

**差异化优势**：GitHub 本身不是为「下载软件」这个场景设计的。它像一个只解决了「发现」但没解决「获取」的应用商店。这个工具补齐了最后一公里——让非开发者也能像在应用商店一样下载和更新开源软件。

## 3. Claude Code 上下文分析器

**核心功能**：追踪 AI 编程助手（如 Claude Code）的上下文窗口使用情况——哪个工具调用消耗最多 token、压缩频率、技能和用户交互各占多少，然后可视化呈现，帮助优化会话效率。

**创意来源**：HN Show 上 manavg76 展示了 context-analyzer 项目。随着越来越多的开发者依赖 AI 编程助手，上下文窗口成为稀缺资源。一个长会话可能因为上下文爆满而丢失关键信息，但用户往往不知道是什么占用了空间。这个工具让上下文消耗变得透明。（[来源](https://news.ycombinator.com/item?id=48471407)）

**技术路径**：读取 Claude Code 的会话日志（JSON 格式），解析 token 用量数据，前端用 Chart.js 或 D3.js 可视化。打包成 npm 全局工具或 VS Code 扩展。纯本地运行，不需要联网。

**差异化优势**：目前 AI 编程助手的上下文管理完全是黑盒——用户只能在碰到上下文限制时被动应对。这个工具把上下文消耗变成可审计、可优化的资源，类似开发者工具的「性能分析器」之于 AI 会话。

## 4. 阅后即焚加密分享

**核心功能**：生成一个加密链接，用于分享密码、API key、日志、敏感文件等。接收方打开后内容自动销毁，发送方能查看是否已被打开。杜绝「密码永远躺在微信聊天记录里」的安全隐患。

**创意来源**：V2EX 用户 heywitt 分享了开源工具 Secret，解决了一个非常具体的痛点——给同事发临时密码或 key，发在微信/QQ 里总觉得不安全，事后还得改密码。工具支持设置过期时间和打开次数限制，内容加密存储，服务器不存明文。（[来源](https://www.v2ex.com/t/1218904)）

**技术路径**：前端 React + 后端 Node.js/Deno。加密逻辑在浏览器端完成（Web Crypto API 生成 AES 密钥），密钥编码在 URL 的 hash 中（不发送到服务器）。服务器只存密文，解密全靠前端。支持设置 TTL 和访问次数上限。可以部署在 Cloudflare Workers 上，几乎零成本运行。

**差异化优势**：对比现有的剪贴板同步或密码管理器，这个工具的定位更轻——不是「管理所有密码」，而是「安全地传一次敏感信息」。不需要注册账号，不需要安装 App，一个链接搞定。这种「用完即走」的极简体验是它最大的护城河。

## 5. Squircle CSS 生成器

**核心功能**：可视化拖拽滑块调整 iOS 风格超椭圆圆角（Squircle）的参数，实时预览效果，一键复制 CSS clip-path 代码或下载 SVG。

**创意来源**：V2EX 用户 aabwold 分享了自己做的纯前端工具 squirclegenerator.com。起因是做 UI 时经常需要 iOS App 图标那种比普通 border-radius 更顺滑的圆角形状，但网上现成工具要么功能受限要么需要登录。（[来源](https://www.v2ex.com/t/1219183)）

**技术路径**：纯前端，HTML + CSS + 少量 JavaScript。核心是超椭圆数学公式（|x/a|^n + |y/b|^n = 1）生成 SVG path 或 CSS clip-path。用 Canvas 实时渲染预览，调节 n 值（2 为普通椭圆，>2 趋近 Squircle，通常 5 左右为 iOS 风格）。部署到 GitHub Pages 或 Vercel，零运维成本。

**差异化优势**：这是一个教科书级别的「微型工具」——功能单一到一句话就能描述清楚，但确实有真实用户需求。前端开发者和 UI 设计师是天然的目标用户。从开发到上线可能只需要一个周末，却能持续获得搜索引擎的长尾流量。
