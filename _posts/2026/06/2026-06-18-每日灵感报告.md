## 1. 免费试用捕手（TrialSnap）

**核心功能**：自动追踪免费试用订阅，在扣费前 24 小时推送提醒，无需绑定银行卡。

**创意来源**：Reddit r/SomebodyMakeThis 上一位用户吐槽："我忘记取消免费试用的次数比我想承认的还要多，结果每次都被扣费。现有的订阅追踪工具（Rocket Money 等）都要求绑定银行账户才能用，我只是想要一个提醒功能而已。"这条帖子直接点出了一个被忽视的痛点——市面上大量「订阅管家」App 功能臃肿，强制绑定财务数据，而用户真正需要的只是一个到期前的通知。来源：[Reddit SMT](https://old.reddit.com/r/SomebodyMakeThis/comments/1u8h066/an_app_that_reminds_you_before_a_free_trial/)

**技术路径**：React Native（跨平台移动端）+ Supabase（后端存储试用记录）+ Firebase Cloud Messaging（推送通知）。甚至可以更精简——做成 PWA + 本地存储 + Web Push API，完全零服务端成本。

**差异化优势**：相比 Rocket Money、Truebill 这类要求绑定银行账户的「全套财务管理」工具，TrialSnap 只做一件事：记录你的试用起止日期，到期前弹一条通知。不碰你的财务数据，不开通任何银行 API，甚至不需要登录——数据存本地，通知走系统原生通道。这才是「用完即走」的工具该有的样子。

## 2. TabLater —— 标签页定时复现

**核心功能**：浏览器扩展，给标签页设置「稍后打开」定时器，到时间自动弹出或提醒你回来查看。

**创意来源**：Reddit r/SomebodyMakeThis 上的一位用户提出了一个反向思路："现有的扩展都是帮你关标签页、限时使用。但我们需要相反的东西——把标签页存起来，在特定时间自动弹回来提醒你处理。"比如：现在看到一篇长文但没空读，设 3 小时后弹回来；看到机票价格，设明天早上复现；收到账单，设下周五提醒。这是一类完全被市场忽略的需求。来源：[Reddit SMT](https://old.reddit.com/r/SomebodyMakeThis/comments/1u6d2jh/tab_reminder_reopens_a_tab_at_a_set_time_opposite/)

**技术路径**：Chrome Extension Manifest V3（支持 Chrome/Edge/Brave）+ IndexedDB 本地存储 + chrome.alarms API（定时触发）+ chrome.tabs API（创建标签页）。纯浏览器端逻辑，零后端依赖。Firefox 版本稍作适配也可支持。

**差异化优势**：现有的「Read Later」工具（Pocket、Instapaper）和书签都是被动存储——存了就忘了。TabLater 是主动提醒：时间到了，标签页自己弹回来。它的核心洞察是「拖延不是因为不想做，而是没在合适的时间被提醒」。相比倒计时番茄钟（限制使用），TabLater 是「正向番茄钟」——帮你把零散想法在合适的时间推回来。

## 3. GitCredit —— 独立开发者的代码贡献热力图

**核心功能**：为任何 Git 仓库生成类似 GitHub 贡献图的绿色热力图，不依赖 GitHub 平台。

**创意来源**：HN Show 上展示的项目 GitCredit，解决了一个小而真实的痛点：很多开发者在使用 GitLab、Gitea、自建 Git 服务器时，缺少 GitHub 那种直观的贡献可视化。独立开发者尤其需要这种「进度可视化」来维持创作动力——绿色的格子就是每天写代码的理由。来源：[HN Show](https://news.ycombinator.com/item?id=48580145)

**技术路径**：前端用 React/Vue 搭配 Canvas/SVG 渲染热力图，后端用 Node.js/Python 调用 `git log` 解析仓库的提交历史。最简 MVP 可以做成纯前端工具——用户拖入仓库路径或粘贴 git log 输出，浏览器端直接渲染。不需要服务器，不需要登录，甚至可以做成 VS Code 插件。

**差异化优势**：GitHub 的贡献图只能看 GitHub 上的活动，而且需要公开仓库或配置特殊规则。GitCredit 的核心价值在于「独立于平台」——不管你的代码托管在哪里，都能获得同一张漂亮的热力图。对于偏好自托管、使用 GitLab 或在国内使用 Gitee 的开发者来说，这是一个被长期忽视的刚需。

## 4. PII Vault —— 发给 AI 前自动脱敏

**核心功能**：本地运行的桌面工具，一键检测文本中的手机号、邮箱、身份证、银行卡等敏感信息，自动替换为占位符后再粘贴到 AI 对话中。

**创意来源**：HN Show 上展示的开源项目 pii-gui 触发了这个思路。原项目是一个通用的 PII 检测桌面应用，但如果聚焦到一个更具体的场景——「每天无数次复制粘贴到 ChatGPT/Claude 中，不知不觉泄露了多少个人信息？」——就变成了一个极具传播力的单点工具。这不是大而全的隐私合规平台，而是每个普通用户每天都能用上的「粘贴前最后一道关卡」。来源：[HN Show](https://news.ycombinator.com/item?id=48579589)

**技术路径**：Electron/Tauri 桌面应用（支持 macOS/Windows），本地调用正则匹配 + 轻量 NER 模型做敏感信息检测，系统级剪贴板监听 + 快捷键触发脱敏。核心逻辑只需几百行代码，难点在于做好交互——比如在粘贴时自动弹出脱敏预览，用户确认后再上屏。甚至可以更轻量：做成 macOS 的 Quick Action 或 Windows 的 Powertoys 插件。

**差异化优势**：现有的数据脱敏工具都是企业级的（需要部署、配置策略），或者是浏览器插件（只能处理网页输入）。PII Vault 定位是一个「粘贴前自动洗一遍」的极简开关——按快捷键、贴出来就是干净的。它不存储任何数据，所有检测在本地完成，完美契合当前「AI 隐私焦虑」的社会情绪。

## 5. PrintLink —— 极简远程打印

**核心功能**：通过一个链接或二维码分享打印机，任何人扫码后上传文件即可打印，无需安装 App。

**创意来源**：V2EX 上一位开发者分享了 PrintHub——一个功能相当完整的云打印平台。但如果我们做减法：砍掉文档社区、砍掉社交功能、砍掉移动端 App，只保留「生成打印链接 → 扫码上传 → 打印机输出」这个核心链路，就变成了一个极其纯粹的微 SaaS 工具。来源：[V2EX 分享创造](https://www.v2ex.com/t/1221331)

**技术路径**：打印机端用 Python/Node.js 写一个小型 Agent（调用系统 CUPS/IPP 打印协议）+ 后端用 Cloudflare Workers（处理文件上传和打印队列）+ 前端一个简单的 Web 上传页。二维码生成直接调用 qrcode.js。整个 MVP 可以在两周内完成，核心代码不超过 1000 行。

**差异化优势**：市面上云打印方案要么绑定品牌（HP Smart、Epson Connect），要么功能臃肿（Google Cloud Print 已死）。PrintLink 的差异化非常清晰——不挑打印机品牌、不需要对方注册、一个链接/二维码就能收文件打印。它的目标用户不是企业，而是：家里有打印机的自由职业者、合租室友之间共享打印机、小团队的公共打印机。本质上它是「打印机的 AirDrop」——简单、即用、不需要解释。
