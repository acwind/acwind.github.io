## 1. Cento —— 5MB 的极简 RSS 文献翻译器

**核心功能**：用 DeepSeek API 自动翻译英文 RSS 标题和摘要，帮助用户快速判断一篇文章是否值得打开原文阅读，整个应用仅 5MB。

**创意来源**：V2EX 用户 scichen 是一位临床医学博士，他在日常阅读英文文献时发现，用 Zotero 管理翻译太笨重，去官网直接读又费劲。于是自己从零学编程，手搓了一个 macOS 风格的轻量 RSS 阅读器，用 AI 翻译标题和摘要。这种"医生跨界写代码解决自己真实痛点"的故事本身就很有启发性——最好的工具往往来自领域内行，而非工程师的空想。

来源链接：[https://www.v2ex.com/t/1220477](https://www.v2ex.com/t/1220477)

**技术路径**：Electron/Tauri 或 SwiftUI（macOS 原生）+ 本地 SQLite 存储 + DeepSeek API（中文翻译质量好且便宜）+ RSS Parser（feedparser）。核心逻辑极简：拉取 RSS → 调用翻译 API 翻译标题和摘要 → 列表展示中英对照 → 点击跳转原文。

**差异化优势**：市面上 RSS 阅读器如 Inoreader、Feedly 要么臃肿要么付费，翻译功能更是需要复杂的 IFTTT 串联。Cento 的思路是"只做一件事"——帮你快速扫一眼英文信息流，翻译成中文，决定要不要细读。5MB 的体积意味着零负担。对于每天需要追踪英文资讯的学术研究者、产品经理、独立开发者来说，这是一个精准切入的小工具。

## 2. Tab Reminder —— 标签页定时复活器

**核心功能**：浏览器扩展，设定一个时间后自动重新打开当前标签页——是"定时关闭标签页"的反向操作，彻底解决"开着一堆标签页不敢关、存书签又永远不看"的痛点。

**创意来源**：Reddit r/SomebodyMakeThis 上的用户 Bocchi_theGlock 发帖吐槽：Firefox 上有一个类似的扩展但锁在付费墙后面，只能设置 20 分钟。而大量用户想要 30 分钟、1 小时、3 小时乃至"明天早上/晚上"这样的灵活选项。帖子引发了大量共鸣——"存书签等于忘记，开着标签页等于焦虑"是当代浏览器用户的通病。

来源链接：[https://www.reddit.com/r/SomebodyMakeThis/comments/1u6d2jh/](https://www.reddit.com/r/SomebodyMakeThis/comments/1u6d2jh/)

**技术路径**：Chrome/Edge/Firefox 浏览器扩展（Manifest V3）+ chrome.alarms API + chrome.tabs API。核心逻辑非常简单：用户点击扩展图标 → 选择预设时间（30分钟/1小时/3小时/明天早上）或自定义 → 扩展关闭标签页 → 到时间后自动在新标签页打开该 URL。数据存储用 chrome.storage.local，零服务器成本。

**差异化优势**：这是一个教科书级别的"单点突破"案例。书签管理器功能臃肿，Pocket 和 Instapaper 是"稍后读"而非"稍后看"，现有的定时器都是"倒计时关闭"而非"倒计时打开"。一个只做"到时间了，帮你重新打开这个页面"的扩展，功能纯粹到可以用一个下午写完第一版。这种反向思维的微工具，往往比大而全的产品更容易获得口碑传播。

## 3. 甄图 —— 极简图片瀑布流筛选器

**核心功能**：在浏览器中打开一个文件夹，以单列瀑布流方式连续浏览所有图片，支持快捷键打标签、分类、筛选——专为需要快速浏览大量截图并做取舍的场景设计。

**创意来源**：V2EX 用户 funet 之前做了一个智能切图工具，最近自己在抓取网站整页截图后发现，这些超长截图在 macOS Finder 里查看非常痛苦——每张都要单独打开、调整大小，想进一步分类和标记更是繁琐。于是用 AI 辅助做了一个浏览器工具，核心交互就是上下滚动翻图片、按几个键打标签。

来源链接：[https://www.v2ex.com/t/1220788](https://www.v2ex.com/t/1220788)

**技术路径**：纯前端 Web 应用（React 或 Vue + File System Access API）+ localStorage 存储标签数据。File System Access API 让浏览器可以直接读取本地文件夹，无需上传服务器。瀑布流布局用 CSS Grid 或简单的 column 布局即可。标签体系支持自定义，导出为 JSON 或 CSV。

**差异化优势**：Adobe Bridge 和 XnView 功能强大但学习曲线陡峭，Eagle 很流行但需要付费且管理逻辑偏重。甄图的定位是"筛选而非管理"——快速过一遍图片，保留或丢弃，标记分类。这种"轻筛选"场景大量存在于设计师选素材、产品经理理截图、摄影师挑片的日常中。不建索引、不建数据库，只是"看 + 标"，用完即走。

## 4. 闲鱼黑话词典 —— 众包搜索词映射工具

**核心功能**：一个众包维基式的查询工具，用户输入想在闲鱼搜索的商品，网站返回能绕过屏蔽的可用搜索词——因为闲鱼对大量合法商品的关键词进行了屏蔽。

**创意来源**：V2EX 用户 vacuitym 早上发现 Codex 需要验证手机号，想去闲鱼买张 giffgaff 卡，结果被关键词屏蔽，试了几个词都不行。于是他做了一个网站 search-sharp.com，允许用户贡献"这个商品能用什么词搜到"的映射关系。这个痛点在中国二手交易场景中极其普遍——闲鱼对 giffgaff、VPN、国际流量等大量正常商品关键词都进行了屏蔽，买家卖家之间形成了一套隐晦的黑话体系。

来源链接：[https://www.v2ex.com/t/1220816](https://www.v2ex.com/t/1220816)

**技术路径**：Next.js 或 Nuxt 全栈应用 + SQLite 或 PostgreSQL + Google OAuth 登录 + 简单的投票或验证机制。核心功能模块：搜索框（输入商品名 → 返回可用搜索词及成功率）、贡献表单（需登录 + 人工或 AI 审核防刷）、热门搜索词排行。可以加上浏览器扩展，在闲鱼搜索页面自动提示替代词。

**差异化优势**：这是一个典型的 UGC（用户生成内容）轻量平台，但切入点极其精准。它不是要做一个大而全的二手交易工具，而是只解决"闲鱼上搜不到"这一个问题。商业上可以走广告或 Affiliate 模式，但作为独立开发项目，核心魅力在于：需求真实且高频（每天无数闲鱼用户都在猜搜索词），网络效应自然（用户贡献内容让平台更有用），开发复杂度可控。

## 5. SharePad —— 一键把 iPad 变成会议白板

**核心功能**：Mac 菜单栏小工具，USB 连接 iPad 后即显示一个干净窗口，可在任何视频会议中共享——无需 QuickTime 的繁琐操作。

**创意来源**：HN Show 用户 jonyardley 每次想在会议上用 iPad 涂鸦时，都要经历"打开 QuickTime → 新建影片录制 → 选 iPad 作为摄像头 → 调整窗口"的繁琐流程。于是他写了一个极简的菜单栏 App——插上 iPad，弹出一个干净窗口，直接在 Zoom、Teams、Meet 里共享即可。目前已有网友评论"这正是我需要的"。

来源链接：[https://sharepad.co/](https://sharepad.co/) | [HN 讨论](https://news.ycombinator.com/item?id=48551113)

**技术路径**：SwiftUI macOS 原生应用 + AVFoundation（捕获 iPad 屏幕作为视频源）+ 系统菜单栏集成。核心逻辑：监听 USB 设备连接 → 检测到 iPad → 启动 AVFoundation 捕获会话 → 渲染到独立窗口。无需网络，纯本地运行，延迟极低。

**差异化优势**：市面上有 Reflector、LetsView 等投屏工具，但它们功能繁多、需要 WiFi、有延迟且多为付费软件。SharePad 的"极简哲学"在于：只做 USB 连接（稳定低延迟）、只显示一个干净窗口（无 UI 干扰）、只是一个菜单栏图标（不占桌面空间）。这种"做减法"的 Mac 小工具在独立开发者社区一直有稳定市场——Dozer、Hidden Bar、Rectangle 都是类似路径的成功案例。
