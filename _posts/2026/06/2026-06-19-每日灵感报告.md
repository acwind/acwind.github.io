## 1. 试用卫士

**核心功能**：自动追踪所有 App/服务的免费试用开始日期，在扣费前 24 小时发送提醒，防止意外的订阅扣款。

**创意来源**：Reddit r/SomebodyMakeThis 上用户 sajith-dev 发帖「An app that reminds you before a free trial charges you」，反映了订阅经济下用户的普遍痛点——注册试用后忘记取消，被默默扣费。帖子虽然评论不多，但点赞比例极高，说明这是大量用户的刚需。

**技术路径**：Swift/SwiftUI（iOS）或 Kotlin（Android），本地 SQLite 存储试用记录，本地推送通知，无需后端服务器。核心逻辑仅为：手动录入试用信息 → 计算扣费日期 → 提前 24h 推送。导入 IAP 收据解析功能可作为 v2 迭代。

**差异化优势**：市面上现有产品（Truebill、Rocket Money 等）功能臃肿，绑定银行卡权限让人不安。试用卫士只需要手动输入试用名称和到期日，不读取任何隐私数据，极简到只有一个列表和一个通知功能。隐私优先、零权限获取信任。

## 2. 标签回响

**核心功能**：让用户设定一个「稍后回来看」的时间，到时自动重新打开该标签页，替代「先放着，等会再看」的心理负担。

**创意来源**：Reddit r/SomebodyMakeThis 上 Bocchi_theGlock 提出「Tab Reminder - Re-Opens a tab at a set time (opposite of a timer to close)」。这个想法的精妙在于它完全反转了常见的「定时关标签」思路——不是帮你关，而是帮你在正确的时间打开。

**技术路径**：Chrome/Edge 浏览器扩展（Manifest V3），chrome.alarms API 定时触发，chrome.tabs API 创建标签页。纯前端实现，零后端。支持设置一次性或重复提醒，可附加简短备注说明「当时为什么要看这个页面」。

**差异化优势**：书签收藏了永远不看，标签页开着占内存又分散注意力。标签回响是二者的精确折中——关掉标签获得心理清爽，但在你选定的时间自动重新呈现。这种「延迟消费」的设计哲学与 Read Later（Pocket）完全不同：Pocket 是你主动去找来看，标签回响是页面主动来找你。

## 3. 场景空间

**核心功能**：macOS 菜单栏工具，一键切换工作/娱乐/会议等预设场景，每个场景自动启动对应 App 组合、调整窗口布局和系统设置（如勿扰模式、音量）。

**创意来源**：V2EX 分享创造板块用户发布「做了个按场景切换 Mac App 的小工具，准备了 30 个兑换码换反馈」，说明作者已经做出了 MVP 并在验证市场需求。

**技术路径**：Swift + macOS Accessibility API 获取窗口控制权限，AppleScript 或 Shortcuts 集成，菜单栏 App 形态。每个场景保存为 JSON 配置，包含 App Bundle ID 列表、窗口位置信息和系统偏好设置指令。

**差异化优势**：Bunch 和 Keyboard Maestro 虽然功能强大但配置学习曲线陡峭。场景空间的目标是「打开即用」——预设工作、写作、会议、娱乐四个模板，用户只需要勾选想启动的 App 就能完成配置。零学习成本的场景切换器，而不是又一本需要读手册的自动化工具。

## 4. 封面快照

**核心功能**：输入 URL，自动抓取网页标题和描述，生成符合 Open Graph 规范的社交分享预览图（OG Image），一键下载或复制。

**创意来源**：V2EX 分享创造板块用户发布「轻量的在线 OG 图生成工具」自荐。OG 图是独立开发者发布产品的必备物料，但目前市面上的工具要么付费（如 Canva），要么需要设计能力（如 Figma），缺少一个纯粹的「输入链接 → 输出封面」的极简工具。

**技术路径**：Next.js 前端 + Puppeteer/Playwright 服务端渲染截图，前端 Canvas/Satori 实时预览生成，支持模板切换（渐变背景、截图 + 标题叠加、纯排版风格）。托管在 Vercel，API Routes 处理后端逻辑。

**差异化优势**：与传统的 OG 图设计工具不同，封面快照的核心思路是「自动化抓取 + 模板化输出」。用户粘贴一篇文章的 URL，工具自动提取标题、作者、描述作为素材填入模板，一键生成 1200x630 的标准 OG 图像。不需要从零开始设计，也不需要学习任何工具。对于经常发布产品的独立开发者来说，这是发布流程中「最后一公里」的效率提升。
