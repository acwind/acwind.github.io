## 1. Vaportrail — AI 编程会话「黑匣子」

**核心功能**：为 Claude Code、Codex 和 OpenCode 等 AI 编程工具提供统一的会话记录与回放，像一个飞行记录仪一样追踪每一次 AI 辅助编程的完整过程。

**创意来源**：HN Show 用户 b33bmo 开发并展示了 Vaportrail，一个专门记录 AI Coding 工具使用历史的 CLI 项目。随着越来越多的开发者使用 AI 辅助编程，对会话的追溯、审计和复盘需求正在快速增长——但各工具之间缺乏统一的记录方案。项目地址：[GitHub](https://github.com/B33BMO/vaportrail)，HN 讨论：[Show HN](https://news.ycombinator.com/item?id=48486482)。

**技术路径**：Rust 或 Go 构建 CLI 核心（保证跨平台和性能），SQLite 本地存储会话数据，支持 JSON 导出。可扩展为 TUI 交互界面用于会话浏览和搜索。

**差异化优势**：现有 AI Coding 工具各自为政（Claude Code 有自己的历史、Codex 有另一套），没有一个统一的"黑匣子"。Vaportrail 补齐了这个空缺，而且完全本地化、隐私友好，不需要依赖任何云服务。

## 2. ClipMemo — 自然语言搜索剪贴板历史

**核心功能**：Mac 上的剪贴板历史管理工具，核心差异在于支持用自然语言描述来搜索历史记录（例如"上周从 Chrome 复制的那个链接"），而不需要记精确的关键词。

**创意来源**：V2EX 用户分享了自己开发这个工具的动机——每天复制粘贴上百次，但剪贴板历史存下来后根本找不到想要的内容。列表拉到几百条靠肉眼翻，搜索又需要精确关键词。用自然语言搜索解决了「我知道我复制过什么，但不知道用什么关键词搜」这个经典痛点。原帖：[V2EX](https://www.v2ex.com/t/1219461)。

**技术路径**：Swift + SwiftUI 构建原生 macOS 应用，使用 Apple 的 Natural Language 框架或本地小模型做语义匹配，CoreData 存储历史条目。可以集成 macOS Accessibility API 监听全局剪贴板变化。

**差异化优势**：市面上剪贴板工具（Paste、Maccy、Clipboard Manager 等）全靠手动翻找或精确关键词搜索，ClipMemo 用语义理解做到「模糊查找」，用户只需描述记忆片段就能定位到目标条目。功能单一纯粹，不做多余的事。

## 3. Refresh — 自托管个人社交信息流聚合器

**核心功能**：一个自托管的 Web 应用，利用浏览器登录态聚合 X/Twitter、知乎、B 站等平台的推荐和关注流，统一到一个干净的阅读界面中。无需 API Key，不依赖平台开放接口。

**创意来源**：V2EX 用户 woodgear 开源了这个项目，思路非常巧妙——通过 CDP（Chrome DevTools Protocol）获取浏览器中已登录的社交平台推荐内容，聚合到一个 API 中。这在技术上规避了平台 API 限制，同时完全保护用户隐私。项目地址：[GitHub](https://github.com/woodgear/refresh)，原帖：[V2EX](https://www.v2ex.com/t/1219487)。

**技术路径**：Node.js/Python 后端 + CDP 协议（Puppeteer 或 Playwright）获取数据，前端用 React 或 Next.js 构建阅读界面。Docker 一键部署，SQLite 存储。建议增加 RSS 输出功能，方便接入任意 RSS 阅读器。

**差异化优势**：与 RSSHub 等通用聚合工具不同，Refresh 是专门针对「个人社交信息流聚合」这个单一场景设计的。不需要配置复杂的路由规则和 API Key，直接用你自己的浏览器登录态——这意味着你看到的就是平台推给你本人的内容，体验和原生 App 一致但更清爽。

## 4. AnyClip — 任意形状网页截图

**核心功能**：一个 Chrome 浏览器扩展，允许用户在网页上自由绘制任意形状（圆形、多边形、路径等）进行截图，突破传统矩形截图的限制。

**创意来源**：V2EX 用户 ns2250225 开发了这款「网页小剪刀」，支持自由绘制截图区域。这个功能场景非常具体——设计师截取 UI 元素、开发者截取不规则区域、内容创作者制作素材时经常需要非矩形截图，但现有工具几乎都只支持矩形框选。项目地址：[GitHub](https://github.com/ns2250225/anyclip)，原帖：[V2EX](https://www.v2ex.com/t/1219492)。

**技术路径**：Chrome Extension Manifest V3 + Canvas API 实现自由路径绘制，利用 `html2canvas` 或 Chrome DevTools Protocol 截取页面，通过 Canvas clip 路径裁剪出最终图片。支持导出 PNG（透明背景）和 SVG。

**差异化优势**：市面上截图工具成百上千，但没有一个专注于「任意形状截图」这个细分场景。AnyClip 只做这一件事，把它做到极致——支持自由路径绘制、透明背景导出、一键复制。功能足够小，体验足够好。

## 5. InputLocker — macOS 输入法智能锁定

**核心功能**：macOS 菜单栏工具，支持全局锁定、按应用锁定、甚至按网址锁定输入法，解决 macOS 频繁自动切换输入法的恼人问题。

**创意来源**：V2EX 用户分享了自己开发的原生 macOS 输入法锁定工具，起因是 macOS 经常莫名其妙切换到 ABC 输入法，严重影响中文输入效率。这个痛点在中英文混用用户中几乎人人都有体会，但现有解决方案要么收费、要么功能不够精准。项目开源在 GitHub，原帖讨论热烈：[V2EX](https://www.v2ex.com/t/1219460)。

**技术路径**：Swift + SwiftUI 构建原生 macOS 应用，通过 Carbon API 或 InputMethodKit 框架监听和切换输入法。菜单栏驻留，支持 Accessibility 权限获取当前前台应用信息。配置以 JSON 文件本地存储。

**差异化优势**：macOS 自带的输入法切换逻辑非常不智能，而现有第三方工具功能臃肿。InputLocker 只做一件事：确保在指定应用/网站中始终使用你指定的输入法。功能极简、配置直观、开源免费——对于每天在多个应用中切换中英文的用户来说，这是一个立竿见影的体验提升。
