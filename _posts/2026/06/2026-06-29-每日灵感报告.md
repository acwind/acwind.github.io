## 1. Chappie — 极简 macOS 桌面启动器

**核心功能**：用不到 3MB 的极简窗口替代 macOS Spotlight，专注做好"快速启动 App 和打开文件夹"这一件事。

**创意来源**：HN Show 上 adamschwartz 展示了 Chappie（[Show HN](https://news.ycombinator.com/item?id=48715445)），它占用 `⌘+SPACE`，支持 App 启动、文件夹导航、`/` 目录穿透、`!` 系统快捷命令、`=` 即时计算，整个二进制仅 2.3MB。目前闭源，计划 1.0 时开源。这个项目的核心洞察是：Spotlight 越来越臃肿、搜索越来越慢，用户只需要一个"瞬间响应"的启动器，而非一个全知全能的搜索框。

**技术路径**：Swift + AppKit 原生开发，macOS 辅助功能权限获取窗口焦点，SQLite 做 App/文件索引缓存，避免每次都扫文件系统。核心逻辑不到 2000 行即可实现。

**差异化优势**：Alfred/Raycast 功能繁多、设置页像控制面板；Spotlight 被 Siri/网页搜索拖慢。Chappie 证明了"够用即可"的哲学——不打算做工作流、插件市场、剪贴板历史，只做启动和导航，体量压制。

## 2. 邮件地址管理器 — 密码管理器思路的轻量延伸

**核心功能**：记住你在每个网站注册时用了哪个邮箱，当收到 Magic Link 或 OTP 验证码时不用逐个邮箱翻找。

**创意来源**：Reddit r/SomebodyMakeThis 上用户提出（[原帖](https://www.reddit.com/r/SomebodyMakeThis/comments/1uieghk/itll_be_like_a_password_manager_but_for_email/)）：密码管理器可以存用户名和密码，但 Magic Link 登录（越来越流行）只发邮件不输密码，此时你需要知道"这个网站我到底用哪个邮箱注册的"。这恰好是 1Password/Bitwarden 都没有优雅解决的问题。

**技术路径**：Chrome/Edge 扩展（Manifest V3），浏览器本地存储无需后端，自动检测登录页面域名，匹配已保存的邮箱记录。MVP 只需一个弹窗 + 域名-邮箱映射表。

**差异化优势**：密码管理器把"邮箱"当作元数据而非主键，搜索体验糟糕；临时邮箱服务（如 10minutemail）解决的是反方向的问题。这个工具只做一件事——让你三秒内看到当前网站关联了哪个邮箱。

## 3. 「先写再说」— 去 AI 味的英文写作纠错工具

**核心功能**：将你写的中式英语（Chinglish）自然、地道地纠正为 native speaker 水准，同时保持原意且不带"AI 翻译腔"。

**创意来源**：V2EX 用户 burden7 开源了这个工具（[原帖](https://www.v2ex.com/t/1223662)，[GitHub](https://github.com/kpaxian7/english_writing)）。独立开发者在 Reddit、X、Product Hunt 上推广产品时，阅读能力尚可但英文书写困难——直接用 ChatGPT 润色又带着浓重的 AI 味（过度正式、滥用 "delve into" "foster" 等词汇），读者一眼识破。

**技术路径**：Node.js/Next.js 网页前端，调用 gpt-4o-mini 等快速模型做纠错（不需要深度推理），关键是写好 system prompt——强调"口语化、自然、保持原意、不添加废话"。支持本地部署，API Key 由用户自行配置。

**差异化优势**：Grammarly 面向母语者做语法检查，DeepL Write 偏正式文体。对于"非母语者想写出自然的日常英文"这个细粒度需求，市面上几乎没有专门工具。它的核心价值在 prompt 设计而非工程复杂度——恰好是独立开发者的优势领域。

## 4. AI 工作状态悬浮指示灯

**核心功能**：在桌面悬浮一个半透明毛玻璃色块，实时反映你的 Claude Code / Codex / Cursor 会话当前状态，扫一眼就知道 AI 在工作、等待、还是已完成。

**创意来源**：V2EX 上两位开发者不约而同做了类似的东西——一个做成了物理硬件红绿灯，另一个做成了网页状态看板（[原帖1](https://www.v2ex.com/t/1223656)、[原帖2](https://www.v2ex.com/t/1223508)）。核心痛点高度一致：同时开多个 AI 会话时，反复切窗口检查进度非常低效，而且等 AI 时容易刷走神，回过神发现 AI 早就跑完了。

**技术路径**：Electron 做桌面悬浮窗（置顶 + 鼠标穿透），WebSocket 连接本地 AI 工具的进程状态。蓝色 = 工作中，红色 = 等待确认，绿色 = 已完成，灰色 = 空闲。MVP 阶段可以先支持 Claude Code 的 CLI 日志解析。

**差异化优势**：相比物理硬件方案零成本、即装即用；相比网页看板方案更直觉——悬浮窗在任意桌面可见，不需要额外设备或切换标签页。这是一个典型的"痒点"工具：没有它也能工作，有了之后回不去。

## 5. Mp2RSS — 把公众号和 X 装进你的阅读器

**核心功能**：将微信公众号和 X（Twitter）账号转换为标准 RSS 源，统一导入任意 RSS 阅读器，告别在微信和 X 之间反复跳转。

**创意来源**：V2EX 用户发布的独立作品（[原帖](https://www.v2ex.com/t/1223619)，[官网](https://mp2rss.bugcode.dev)）。在中国互联网生态中，微信公众号是内容创作的主阵地，但微信阅读体验极差——无法标记已读、无法分类、无法稍后阅读、无法全文搜索。而 RSS 恰好能解决这些问题，只是微信从未提供官方 RSS 接口。

**技术路径**：Python/Node.js 后端做公众号内容抓取和解析，生成符合 RSS 2.0 标准的 XML Feed。支持 Web、CLI、OpenAPI 和 AI Agent 多种访问方式。关键技术难点在反爬和稳定性，但核心逻辑并不复杂。

**差异化优势**：类似的 WeRSS、RSSHub 要么不够稳定、要么配置门槛高。Mp2RSS 的定位是"开箱即用"——输入公众号名称或 X 用户名，直接返回 RSS 链接。这个细分市场虽然小众，但用户付费意愿强（信息获取效率是刚需）。
