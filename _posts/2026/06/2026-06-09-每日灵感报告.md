## 1. DaysLeft — 生物年龄时钟

**核心功能**：基于生活方式数据估算你的"生物年龄"，展示为一个区间（而非精确的死亡日期），让你直观感受生活习惯对健康的影响。

**创意来源**：HN Show 上 neo-genesis 展示的 [daysleft.io](https://daysleft.io)，获得 2 分 1 评论。核心理念是"不告诉你何时会死，而是告诉你现在的身体年龄"，这种去焦虑化的设计取向非常巧妙。相比市面上的寿命预测器，它强调"范围"而非"日期"，传递的是希望感而非恐慌感。

**技术路径**：前端用 React/Next.js 构建问卷式 UI，后端用 Python Flask 搭建简单的健康模型 API（可基于公开的 Framingham 或 UK Biobank 数据集做粗糙估算），数据存 SQLite。MVP 就一个单页问卷 + 结果页，完全可以在 1 周内完成。

**差异化优势**：市面上的健康 App 要么过于复杂（记录饮食/运动/睡眠），要么走向玄学（算命式寿命预测）。这款产品只做一件事：用 10 个简单问题（吸烟、运动频率、睡眠时长、BMI 等）给出一个生物年龄区间，极简入口，无负担使用。

## 2. Secret — 阅后即焚的临时分享工具

**核心功能**：安全地发送临时密码、密钥、日志片段或小文件给他人——内容端到端加密存储在服务端，过期或阅后自动销毁，发送方可查看是否已被打开。

**创意来源**：V2EX 用户 [分享的 Secret 项目](https://www.v2ex.com/t/1218904#reply11)，已在 GitHub 开源。用户反馈痛点真实："给别人发临时密码/API Key/日志，丢微信里总觉得会在各种地方躺很久，被迫去改密码。" 支持 Cloudflare 部署，预估日活万次内月成本接近零。

**技术路径**：前端用简单的 HTML/JS（或 Vue/React），后端用 Cloudflare Workers + KV 存储，加密算法用 Web Crypto API（AES-GCM）。核心逻辑不到 500 行代码，2-3 天即可完成 MVP。可一键部署到 Cloudflare Pages。

**差异化优势**：相比 1Password/LastPass 的分享功能（需双方注册），或 Pastebin（永久公开），Secret 专注做"临时 + 加密 + 阅后即焚"这一个点。无账号体系、无追踪、开源可自部署，对于注重隐私的开发者群体极具吸引力。

## 3. Window Cycle — 当前 App 窗口精准切换器

**核心功能**：macOS 菜单栏小工具，用 Cmd+` 在**当前最前台应用**的多个窗口之间循环切换，比系统自带的窗口切换更直观、更精准。

**创意来源**：V2EX 用户 [分享的 window-cycle 项目](https://www.v2ex.com/t/1218880#reply10)，GitHub 开源。作者精准描述痛点："同一个 IDE、浏览器、Terminal 开了很多窗口，Cmd+Tab 是切 App 不够精确，系统自带的窗口切换又比较隐形。"

**技术路径**：Swift + AppKit 开发 macOS 菜单栏应用，核心功能依赖 Accessibility API 获取当前前台应用窗口列表，注册全局快捷键。整个工具逻辑极简——获取窗口列表、切换焦点窗口，1 周内可完成并上架。

**差异化优势**：市面上有 Contexts、AltTab 等窗口管理工具，但它们功能大而全、设置复杂。Window Cycle 只做一件事：在当前 App 内切换窗口，零配置、无学习成本。这种"减法思维"是独立开发者最稀缺的产品直觉。

## 4. Playlist Bridge — 跨平台歌单同步器

**核心功能**：将 Spotify 歌单与 YouTube Music 双向同步，自动匹配曲目。解决"朋友分享 Spotify 链接，但你是 YouTube Music 用户"的日常尴尬。

**创意来源**：Reddit SMT 上 ThemaskedGamerAS 提出的 [Blend playlist across Youtube and Spotify](https://old.reddit.com/r/SomebodyMakeThis/comments/1tz5qg3/blend_playlist_across_youtube_and_spotify/)。这个痛点极为普遍——音乐流媒体平台割裂，朋友间分享歌单总是遇到"我没有这个平台"的困境。

**技术路径**：Python/Node.js 后端，同时接入 Spotify Web API 和 YouTube Data API v3。曲目匹配可用"歌名 + 艺人名"做模糊搜索，辅以 ISRC 码精确匹配。前端可以就是一个粘贴链接 → 选择目标平台 → 返回结果的三步流程页。5-7 天可完成 MVP。

**差异化优势**：Soundiiz 和 TuneMyMusic 已占据这个赛道，但它们收费且功能臃肿（支持 40+ 平台同步）。Playlist Bridge 可以聚焦 Spotify ↔ YouTube Music 这一对最高频的场景，免费、无需注册、三步完成，用极致精简打动用户。

## 5. PersistQ — 防刷新防崩溃的浏览器排队系统

**核心功能**：一个纯前端的排队/队列页面——用户加入排队后，即使刷新页面、浏览器崩溃、甚至换设备，只要用同一个标识（如手机号/邮箱），就能恢复到原来的排队位置。

**创意来源**：Reddit SMT 上 fitzerb18 提出的 [A ticket queue that survives refreshes, crashes, and switching devices](https://old.reddit.com/r/SomebodyMakeThis/comments/1u0anto/a_ticket_queue_that_survives_refreshes_crashes/)。场景极为具体：抢演唱会门票、医院挂号排队、限量商品购买——现有的购票排队系统体验糟糕，一刷新就回到队尾。

**技术路径**：前端 React + IndexedDB，后端 Node.js + Redis（队列状态持久化）。核心机制是：用户加入队列时分配唯一 token 并存入 localStorage/后端 KV，前端断连后自动用 token 恢复位置。心跳机制保活 + 断线重连，7-10 天可完成 MVP。

**差异化优势**：Queue-it 是企业级方案收费高昂，且过度复杂。PersistQ 专注做"轻量级的可恢复排队"这一单点——适合中小型活动主办方、社区抽签、限量发售等场景，可以做成 SaaS 或开源自部署。
